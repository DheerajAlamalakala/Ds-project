Circular Linked List

Same as a linked list, except the last node's pointer loops back to the head instead of stopping at NULL.

struct Node {
    int data;
    struct Node* next;
};

struct Node* head = NULL;

void insertTail(int val) {
    struct Node* n = malloc(sizeof(struct Node));
    n->data = val;
    if (head == NULL) { n->next = n; head = n; return; }
    struct Node* cur = head;
    while (cur->next != head) cur = cur->next;
    cur->next = n;
    n->next = head;
}

void deleteValue(int val) {
    if (head == NULL) return;
    struct Node *cur = head, *prev = NULL;
    do {
        if (cur->data == val) {
            if (prev == NULL && cur->next == head) { head = NULL; return; }
            if (cur == head) head = cur->next;
            prev->next = cur->next;
            return;
        }
        prev = cur; cur = cur->next;
    } while (cur != head);
}

How this works

The only structural difference from a regular linked list is what the last node's next points to: instead of NULL, it points straight back to head. There's no "end" to find — which means traversal loops have to check cur->next == head to know when they've gone all the way around, not cur->next == NULL.

Insert at tail walks until it finds the node whose next currently equals head — that's the last node — then re-points it to the new node, and the new node's next takes over pointing at head.

This shape is genuinely useful for things like round-robin scheduling or a music playlist set to repeat — anything that needs to cycle back to the start without an explicit "end" case.
