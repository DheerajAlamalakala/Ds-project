Tree Sort

Insert every value into a BST, then read it back out with an inorder traversal — sorted, for free.

struct Node* insert(struct Node* root, int val) {
    if (root == NULL) {
        struct Node* n = malloc(sizeof(struct Node));
        n->data = val;
        n->left = n->right = NULL;
        return n;
    }
    if (val < root->data) root->left = insert(root->left, val);
    else root->right = insert(root->right, val);
    return root;
}

void inorder(struct Node* root, int* out, int* k) {
    if (root == NULL) return;
    inorder(root->left, out, k);
    out[(*k)++] = root->data;
    inorder(root->right, out, k);
}

void treeSort(int arr[], int n) {
    struct Node* root = NULL;
    for (int i = 0; i < n; i++)
        root = insert(root, arr[i]);
    int sorted[n], k = 0;
    inorder(root, sorted, &k);
    // sorted[] now holds arr in ascending order
}

How this works

Tree Sort doesn't have a sorting step of its own — it just builds an ordinary BST by inserting every array value one at a time, then leans on a property that's already true of any BST: an inorder traversal (left, node, right) always visits values in ascending order.

The cost is hidden in the build phase. Each insert is O(log n) on a balanced tree but degrades to O(n) on a skewed one (e.g. inserting already-sorted data), which is why Tree Sort's worst case matches the slower comparison sorts despite looking different on the surface.
