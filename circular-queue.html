How this works

        

          A plain queue wastes space — once front moves past index 0, those early slots
          never get reused. The circular queue fixes this by treating the array as a ring:
          every index calculation wraps using the modulo operator, %.
        

        

          Enqueue advances rear with rear = (rear + 1) % MAX —
          so once rear hits the last index, the next enqueue sends it straight back to 0. It's
          considered full not when every slot is physically used, but when advancing rear would
          land exactly on front again: (rear + 1) % MAX == front.
        

        

          Dequeue works the same way in reverse: front = (front + 1) % MAX.
          Watch the pointer labels under the ring — once they reach the rightmost slot, the next operation
          jumps them back to the leftmost slot instead of running off the end.
