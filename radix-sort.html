void countingSortByDigit(int arr[], int n, int place) {
    int output[n];
    int count[10] = {0};

    for (int i = 0; i < n; i++)
        count[(arr[i] / place) % 10]++;

    for (int i = 1; i < 10; i++)
        count[i] += count[i - 1];

    for (int i = n - 1; i >= 0; i--) {
        int digit = (arr[i] / place) % 10;
        output[count[digit] - 1] = arr[i];
        count[digit]--;
    }

    for (int i = 0; i < n; i++) arr[i] = output[i];
}

void radixSort(int arr[], int n) {
    int max = getMax(arr, n);
    for (int place = 1; max / place > 0; place *= 10)
        countingSortByDigit(arr, n, place);
}

How this works

Radix Sort never compares two numbers directly. Instead, it processes one digit place at a time — ones first, then tens, then hundreds — and on each pass distributes every number into one of 10 buckets based on just that digit, then collects the buckets back into the array in order 0 through 9.

The key detail is stability: within a bucket, numbers must stay in the relative order they arrived in. That's why each pass uses a stable counting sort rather than just dumping values in — if an earlier pass already sorted the lower digits correctly, an unstable later pass would scramble that work.

After processing the most significant digit of the largest number, the array is fully sorted — the digit place loop in radixSort simply stops once place exceeds every number's value.

CaseTime
All casesO(d \u00B7 n)
SpaceO(n + 10)

d = number of digits in the largest value. For fixed-width integers, this is effectively O(n).
