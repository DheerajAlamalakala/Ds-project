How this works

        

          A queue keeps two pointers into the same array: front, the oldest element still
          waiting, and rear, the most recently added one. Both start at -1 to
          mean "empty" — there's no valid index yet.
        

        

          Enqueue checks there's room (rear == MAX - 1), then if the queue
          was empty it sets front = 0 so it now points at the very first slot. Either way it
          advances rear and writes the new value there: arr[++rear] = val.
        

        

          Dequeue reads arr[front] as the value leaving. If front
          and rear meet at the same index, the queue is now empty and both reset to
          -1. Otherwise front simply moves forward by one. Notice the slots
          behind front are never reused in this simple version — that's the gap a
          circular queue is built to close.
