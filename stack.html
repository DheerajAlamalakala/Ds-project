How this works

        

          The stack is backed by a fixed-size array arr[MAX] and a single integer
          top, which always points to the index of the most recently inserted element.
          An empty stack starts with top = -1 — there's nothing to point to yet, so every
          slot in arr still holds whatever garbage value was sitting in memory before the
          program touched it.
        

        

          Push first checks top == MAX - 1 to make sure there's room left.
          If there is, it increments top and writes the new value at that index in one step:
          arr[++top] = val. The pre-increment means "move the pointer up first, then write" —
          so the value always lands one slot above whatever was previously on top.
        

        

          Pop checks top == -1 to make sure the stack isn't empty, then reads
          arr[top] and decrements top in the same line:
          return arr[top--]. The data technically stays in memory at that array index — it's
          just no longer considered part of the stack, since top no longer points to it. That's
          why a popped slot is shown as garbage again in the visualizer.
